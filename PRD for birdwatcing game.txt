《观鸟日记》产品需求文档（PRD）v4.0（完整叙事版）
1. 文档信息
产品名称：观鸟日记

版本：4.0（完整叙事+分阶段挑战）

更新日期：2026-06-08

产品负责人：独立开发者

2. 产品概述
2.1 定位
一款沉浸式叙事驱动的观鸟科普小程序/HTML5游戏。玩家以“观鸟爱好者”或“巡护体验官”身份进入芦苇湿地保护区，通过望远镜观察、认鸟挑战、生态事件推理等环节，逐步体验自然巡护工作，学习鸟类知识，并参与反盗猎保护行动。

2.2 核心价值
叙事沉浸：以跟随巡护员老周的视角，构建完整故事线，增强代入感。

双身份选择：自由观鸟（放置模式）与巡护挑战（任务制）满足不同玩家偏好。

分阶段学习：认鸟测试分为普通鸟、保护鸟类、亚成鸟三个阶段，难度递增，强化识别能力。

生态推理：通过残骸位置、捕食关系分析，传递自然食物链与反盗猎意识。

2.3 目标用户
观鸟初学者（希望系统学习鸟类识别）

自然教育爱好者（亲子、学生群体）

轻度游戏玩家（喜欢任务引导和叙事）

3. 游戏流程与叙事设计
3.1 开场动画（双筒望远镜视角）
视觉：黑色双圆形遮罩（模拟望远镜目镜），透过圆环可见芦苇荡动态背景（风吹芦苇、飞鸟剪影）。

动效：轻微呼吸式晃动，边缘光晕，持续3秒后淡出。

交互：可点击跳过。

3.2 故事引入（字幕或对话框）
文本：

“清晨的薄雾还未散尽，我们跟随老周——这片芦苇荡的巡护员，踏进湿地深处。他在这里守了二十年，认得每一种鸟的叫声，也见过偷猎者留下的网。今天，他将带着我们几个观鸟爱好者，走一趟巡护路。”

交互：点击屏幕逐句显示，最后出现“身份选择”按钮。

3.3 身份选择界面
身份	图标	玩法模式	核心体验
🌿 观鸟爱好者	望远镜	放置模式（自由观鸟，无任务压力）	放松、收集、科普
🚩 巡护体验官	臂章	挑战模式（分阶段任务）	学习、推理、保护行动
选择后，游戏根据身份加载不同初始场景，并在界面右上角显示身份徽章。

4. 巡护体验官模式详细设计（分阶段挑战）
4.1 总体任务线
巡护员老周将作为向导，分三个阶段发布任务。每个阶段完成后方可解锁下一阶段。

阶段	主题	核心任务	时长预估
第一阶段	认识普通鸟类	正确识别3种芦苇湿地常见鸟（如大白鹭、震旦鸦雀、东方大苇莺）	5-10分钟
第二阶段	认识保护鸟类	识别2种国家重点保护鸟类（如白琵鹭、黑脸琵鹭、东方白鹳等，依地区而定）	10-15分钟
第三阶段	认识亚成鸟	识别2种常见鸟的亚成鸟（如夜鹭亚成、白鹭亚成，与成鸟区别明显）	10-15分钟
4.2 任务交互细节
第一阶段：普通鸟类识别
触发：老周对话框：“这片芦苇荡最常见的几种鸟，你能认出它们吗？”

玩法：场景中随机刷新3种普通鸟（各一只），玩家需双击每只鸟并从选项中选择正确名称。

反馈：答对时老周夸奖；答错时显示正确科普并增加该鸟认错计数（后续权重提高）。

完成奖励：生态积分+30，解锁“普通鸟类爱好者”徽章。

第二阶段：保护鸟类识别
触发：老周：“瞧，那边有几只珍稀鸟，国家重点保护，你能认出来吗？”

玩法：刷新2种保护鸟类（可能混入普通鸟干扰），玩家需要正确识别并点击收集。

反馈：答对后弹出保护级别科普（如“国家二级保护动物”），并提示“请保持距离，不要惊扰”。

完成奖励：生态积分+50，解锁“护鸟使者”徽章。

第三阶段：亚成鸟识别
触发：老周：“那些羽色暗淡的可不是别的鸟，是今年刚长大的小朋友。你能认出它们是谁家的孩子吗？”

玩法：刷新2种亚成鸟（与成鸟外观差异显著，如夜鹭亚成棕褐色带斑点），玩家需正确选择对应的成鸟名称。

反馈：答对后展示成鸟与亚成鸟对比图，说明成长过程。

完成奖励：生态积分+80，解锁“鸟类学家学徒”徽章。

4.3 额外生态事件融入（不强制）
在任意阶段进行中，可能随机出现自然残骸或盗猎线索（概率较低）。

玩家可主动点击并按照之前设计进行推理/报警，获得额外积分且不影响主线任务进度。

4.4 结局与循环
完成三个阶段后：老周对话：“你已经能独立巡护了，以后这片湿地交给你了。”

解锁：自由挑战模式（可重复生成随机组合的认鸟任务）以及“湿地守护者”永久成就。

新游戏+：可重置任务进度重复体验，或继续自由放置观鸟。

5. 界面与交互设计要点
5.1 开场动画参数
遮罩：两个圆形半径100px，圆心分别在屏幕宽度的35%和65%处。

背景：芦苇荡动态GIF或循环动画（可使用Canvas绘制简单芦苇摆动）。

晃动：每帧随机偏移±2px，平滑插值。

5.2 故事文本显示
使用半透明黑色底框，白色文字，居中底部。

逐字打印效果（每0.05秒显示一个字）。

点击任意位置显示下一句，最后一句点击后进入身份选择界面。

5.3 身份选择界面
两个卡片样式，左蓝右绿，大号图标。

鼠标/手指悬停放大效果。

选择后淡出并进入对应游戏模式。

5.4 巡护任务面板
右上角固定显示老周头像和当前任务描述（可折叠）。

任务进度条显示三个阶段完成百分比。

完成阶段时弹出成就通知（底部滑入）。

7. 非功能需求补充
场景切换流畅：开场→故事→身份选择→游戏模式之间切换使用淡入淡出，时长0.3秒。

任务提示：若玩家长时间（>2分钟）未完成阶段内任务，老周给出文字提示（如“试试双击那只白鹭”）。

亚成鸟识别难度：亚成鸟图片应与成鸟有明显区别，确保基于视觉可辨别（不依赖文字提示）。

8. 开发优先级
P0：开场望远镜动画 + 故事文本 + 身份选择（预计2小时）

P0：巡护体验官三阶段任务框架（复用现有挑战模式识别逻辑，增加阶段切换和存储）

P1：保护鸟类与亚成鸟专属图标/照片（需额外美术素材）

P2：随机生态事件与任务线自然融合（可后续添加）

《观鸟日记》程序设计文档（补充：叙事与分阶段任务模块）
1. 模块划分新增
1.1 故事与身份管理模块（storyManager.js）
职责：控制开场动画、故事文本播放、身份选择UI及状态存储。

主要函数：

playOpeningAnimation()

showStoryText(textArray, onComplete)

showIdentitySelection(callback)

setIdentity(identity)

1.2 巡护任务模块（rangerQuest.js）
职责：管理三个阶段的任务生成、进度追踪、奖励发放。

主要数据结构：

javascript
const questStages = [
  { name: "普通鸟类", birdIds: ["heron_great", "reed_parrotbill", "reed_warbler"], rewardPoints: 30, badge: "普通鸟类爱好者" },
  { name: "保护鸟类", birdIds: ["spoonbill", "stork"], rewardPoints: 50, badge: "护鸟使者" },
  { name: "亚成鸟", birdIds: ["night_heron_juvenile", "egret_juvenile"], rewardPoints: 80, badge: "鸟类学家学徒" }
];
主要函数：

startQuestStage(stageIndex)

onBirdIdentified(birdId) – 检查当前阶段需要的鸟是否被正确识别

advanceToNextStage()

completeRangerCampaign()

1.3 与现有挑战模式的兼容
当 selectedIdentity === "ranger" 时，挑战模式不再随机生成鸟类，而是调用 rangerQuest.startQuestStage(currentStage) 生成固定鸟种列表。

识别逻辑复用现有的showChallengeIdentify及权重更新，但完成后调用onBirdIdentified。

2. 状态机扩展
添加 gamePhase 状态：

OPENING → STORY → IDENTITY_SELECT → GAME_IDLE / GAME_RANGER

3. 美术资产需求补充
望远镜遮罩圆形光晕（透明PNG）

老周头像（手绘半身）

保护鸟类徽章、巡护臂章图标

亚成鸟与成鸟对比图（弹窗用）

4. 技术实现风险与对策
亚成鸟识别准确率：依赖清晰的手绘差异，建议在素材准备时重点突出羽色、斑纹、嘴型等特征。

任务进度持久化：需将 rangerProgress 保存到 localStorage，确保刷新页面不丢失。

### 新增功能 1：时间与季节系统

**位置：** `autoload/GameTimeManager.gd`

- 游戏内时间从早上6:00开始，以8倍速实时流逝
- 每7天换一季：春→夏→秋→冬，四季循环
- 时间影响鸟类出现（候鸟按季节迁移）
- 季节显示在 HUD 时间面板（TimePanel2）
- 速度可在 4x / 8x 之间切换

**关联数据：** `birds.json` 中每个鸟种有 `migrant` 字段（留鸟/夏候鸟/冬候鸟/过境鸟/迷鸟）

---

### 新增功能 2：6层精灵组系统（背景分层）

**位置：** `scripts/scenes/wetland/layer_manager.gd` + `sprite_group.gd`

场景背景分为6层，每层独立控制透明度：

| 层级 | 名称 | 内容 |
|------|------|------|
| L1 | 🏞️ 远景组 | 天空、远山、树梢 |
| L2 | 🌲 中景组 | 树木、灌木（待填充素材） |
| L3 | 🌿 近景组 | 芦苇杆、近处植物 |
| L4 | 💧 水体组 | 水面、睡莲 |
| L5 | 🌸 前景装饰组 | 地面、花朵、前景芦苇 |
| L6 | 天空层_Sky | 纯色天空背景 |

---

### 新增功能 3：植物摇摆着色器

**位置：** `shader/plant_sway.gdshader`

- 芦苇和树枝顶点动画，底部（根）稳定、顶部（叶）自然摆动
- 参数：`sway_speed`（速度）、`sway_amount`（幅度）
- 远景组应用 speed=3/amount=11，近景组 speed=2/amount=10
- 测试场景：`scenes/sway_test.tscn`
